Version 3.22.2

Release Date August 14, 2013

Maintenance release/Updates/Add-Ons

  • Update: Allow rewrite_rules to be updated when saving settings.

  • Bug: Dolphin theme was sometimes pulling in wrong price value for was/now.

Upgrading to 3.22.2

  • Go to the Settings page within the DealerTrend API Plugin's options page. Run the Clean Uninstall procedure. This will clean up the database and deactivate the plugin.

  • Delete ALL of the old plugins files. It will be /wp-content/plugins/dealertrend-inventory-api. Replace with new files and put them into a folder called dealertrend-inventory-api.
  • Alternatively you can use the automatic updater.

Upgrading from 3.6.0 to 3.7.0

  • Go to the Settings page within the DealerTrend API Plugin's options page. Run the Clean Uninstall procedure. This will clean up the database and deactivate the plugin.
    • This will not delete the plugin. You still need to do that. You can do it from the plugin management screen or via FTP.
  • Delete ALL of the old plugins files. It will be /wp-content/plugins/dealertrend-inventory-api. Replace with new files and put them into a folder called dealertrend-inventory-api.
  • Alternatively you can use the automatic updater.

  • Resolved fundamental errors of last program including frequent database writes.
  • Adding helper to integrate with the VMS API.
  • Creation of the Armadillo Theme.
  • Creation of the Bobcat Theme.
  • Adding SEO Helpers functionality.
  • Adding auto-updater that's intended to pull updates from GitHub.
  • Creation of options screen app/views/wp-admin/options.php.
  • Implementing crude namespacing.
  • Creating HTTP API Wrapper @ app/helpers/http_api_wrapper.php.
  • Took advantage of the built in Wordpress caching API.

Upgrading from 2.x (or prior) to 3.0.0

  • Delete ALL of the old plugins files. It will be a folder within /wp-content/plugins/. Replace with new files and put them into a folder called dealertrend-inventory-api.
  • Note: The old plugin did not have a feature to clean up the database. So there will be traces of the old plugin in your database unless you manually clean them out.
    • This is not a big deal and should not hurt anything in your WordPress install.
